Acceptance + implementation

  • Top nav gains ONE new entry — Assessments — positioned between Surveys and Files (suggested) for visual proximity to the existing form-tooling cluster. Mobile-responsive via the existing overflow-x-auto pattern on (dashboard)/layout.tsx.
  • Existing 13 nav items remain unchanged: Dashboard, All Workshops, Bio, Templates, Workflows, Surveys, Files, Partners, Coaches, Approvals, Categories, Pricing, Financials.
  • Click on Assessments → route changes to /assessments (or /admin/assessments for admin contexts). The dashboard layout renders the assessment-lane sidebar via a child layout file at src/src/app/(dashboard)/assessments/layout.tsx. Sidebar contains: Dashboard, Organizations, Access Groups, Templates, Campaigns, Public Quizzes, Aggregate Report — plus a Coach-lane group below (My Campaigns, My Organizations) shown when the actor is a coach.
  • Existing workshop platform routes do NOT render a sidebar; the assessments lane is the first dashboard route to introduce one. Visually parallels the coach portal's (portal) sidebar but uses the assessment-lane nav set instead.
  • Role gating: top nav Assessments entry visible to ADMIN + STAFF + COACH (lane content differs by role); Aggregate Report sidebar entry visible only to ADMIN + STAFF (canAccessAggregateReport); Coach-lane group visible only to COACH.
Wireframe end note. Chrome only — this is a NAVIGATION integration sketch, not a functional screen.
