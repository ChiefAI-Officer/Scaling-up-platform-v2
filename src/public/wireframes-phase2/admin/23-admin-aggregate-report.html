What's NOT in v1 MVP: time-range filter, AccessGroup filter, per-organization breakdown table, cross-version comparison view, peer benchmark overlays. All deferred to v1.5. The current dashboard returns frozen stats from getAggregateReport(templateId, versionId) across ALL orgs and ALL time per locked decision 8.

v7.6 service-layer surface

  • getAggregateReport(templateId, versionId) → returns frozen stats: totalSubmissions, distinctOrgs, avgCountAchieved, avgOverallTotal, avgOverallAverage, tierHistogram, perSectionMeans, submissionsOverTime.
  • canAccessAggregateReport(actor, templateId) → admin/staff only. Coach actors return false; route returns 403.
  • Version-boundary contract: AssessmentTemplateVersion is the aggregation unit. Per-section means and countAchieved only compare apples-to-apples within ONE immutable version. compareVersions(versionA, versionB) is v1.5; cross-version overlays are NOT in v1.
  • Observability gate: assessment.aggregate.query.duration_ms p95 > 2000ms triggers a paged warning per docs/specs/v7.6/06-observability.md. v1 scale (<500 submissions / template / version) stays well under that. Beyond v1.5, a materialized view backs the query.
Wireframe end note. v1 MVP shape per locked decision 8: TWO selectors (template + version), no filters, no placeholder cards for v1.5 features. The version dropdown defaults to the latest published version and badges it as CURRENT. Aggregation is per AssessmentTemplateVersion, NOT per template across versions — this preserves the apples-to-apples invariant for per-section means. Future v1.5 enhancements (group filter, time range, per-org breakdown, cross-version comparison) land on the same route; they do NOT belong in v1.
