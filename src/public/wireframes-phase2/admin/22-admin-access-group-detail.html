v7.6 service-layer surface

  • canAccessTemplate — INTERSECTION across all of the coach's non-archived group memberships. Per 02-service-layer-rules.md: zero non-archived groups → zero access; one missing template in any group → that template excluded.
  • evaluateAccessChange(tx, coachId, proposedGroupIds) — transactional guard. Runs in one Prisma $transaction at SERIALIZABLE isolation; acquires pg_advisory_xact_lock on access-change:<coachId> AND access-group:<groupId> in alphabetical order; SELECT … FOR UPDATE on the group + every membership/template join in scope; re-reads the affected-coach list inside the transaction (Round 3 H-4); writes AuditLog transactionally via tx.auditLog.create (NEVER the swallowing logAudit() wrapper).
  • archiveAccessGroup(groupId) / undeleteAccessGroup(groupId) / hardDeleteAccessGroup(groupId) are all gated by the same guard. Archive shrinks effective access (intersection); undelete can ALSO shrink under INTERSECTION (an extra constraining group is added back).
  • canManageCampaign — coach who loses template access keeps READ on existing campaigns; cannot invite new respondents or create new campaigns with that template. Active campaigns stay live.
  • BLOCKED_ZERO_ACCESS → HTTP 409. force=true in the request body bypasses the guard and writes AuditLog.action="FORCE_ZERO" with a mandatory reason field.
