Scoring Configuration

How responses convert into a headline metric and tier message.

Drives the headline metric the tier ranges resolve against.
A question counts as “achieved” when its score ≥ this value. Rockefeller uses 2 (on a 0–3 scale).

Tiers

Each tier defines a metric range + label + message shown on the results page. v1: tiers must cover [0, maxMetric] with no gaps or overlaps (Zod refine enforces this on save).

Validation rules (Zod refine on save)

  • All tiers’ [minMetric, maxMetric] ranges must cover [0, maxPossibleMetric] (here maxPossibleMetric = 40 for Rockefeller’s 40 questions × passThreshold).
  • No gaps allowed between consecutive tiers.
  • No overlaps allowed.
  • maxMetric of the last tier may be omitted (treated as infinity).

Conditional Sections

Dynamic report copy. Each section evaluates a when clause against the respondent’s answers + computed result; if true, the section’s markdown content renders in their report.

Coming in v1.5
For v1, admins seed conditionalSections JSON via Prisma Studio. Scaling Up Assessment ships placeholder sections; Jeff replaces with real content via this UI in v1.5. Renderer-side evaluation (<AssessmentResultView> reading the seeded JSON) ships in v1 — only the admin authoring UI is deferred.

Example (read-only)

{
  "stableKey": "employee-count-paragraph-mid",
  "sortOrder": 1,
  "when": { "stableKey": "Q_employee_count", "op": "between", "value": [50, 149] },
  "markdownContent": "## Companies your size (50–149 employees)\n\nAt this stage, the most common bottleneck is..."
}

Why is this section deferred? (Codex co-validate, May 12 2026)

The original v7.5 draft put the conditionalSections (when-clause builder) and peerBenchmarks value editors on this Scoring & Tiers screen. The Codex peer review flagged the when-clause builder as the highest-complexity Wave 2 screen — a mini rule-engine UI with operator selection, value-type variance, and version-aware stableKey resolution. Scaling Up Assessment’s content matrix isn’t ready anyway (Jeff still authoring tier messages for scores 7 & 10), so the renderer-side evaluation lands in v1 reading seeded JSON, and the admin authoring UI ships v1.5. This frees ~1–2 days off the June 15 critical path.

Wireframe end note. Scoring + Tiers tab is the editable v1.5 surface. Conditional Logic tab (currently disabled in the tab nav) becomes the dedicated home for conditionalSections + peerBenchmarks editors when they ship in v1.5.
