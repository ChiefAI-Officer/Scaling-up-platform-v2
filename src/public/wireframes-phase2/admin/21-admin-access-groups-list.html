v7.6 service-layer surface

  • AccessGroup model + AccessGroupCoach + AccessGroupTemplate join tables drive group membership and template association.
  • canAccessTemplate(coachId, templateId) evaluates INTERSECTION across all of the coach's non-archived group memberships. Returns false if the coach is in zero groups OR if any group lacks the template.
  • evaluateAccessChange(tx, coachId, proposedGroupIds) previews before/after template sets for affected coaches when admin adds/removes a coach or template. Used by Wireframe 22 preview modal. Runs at SERIALIZABLE isolation with advisory locks (per 02-service-layer-rules.md).
  • Runtime flag ACCESS_POLICY_VERSION (current: v1.intersection) lets us swap to shadow-union (canary) or union (emergency revert) without a code deploy.
  • archiveAccessGroup / undeleteAccessGroup / hardDeleteAccessGroup all gated by the same evaluateAccessChange guard — archiving shrinks effective access; the guard intercepts shrink-to-zero unless admin passes force=true.
Wireframe end note. The + New Access Group button is admin-only (checked via isPrivilegedRole); coaches NEVER reach this page. Clicking a group's name or the › chevron navigates to Wireframe 22 (admin/access-groups/<id> detail view) which includes the membership tables AND the evaluateAccessChange preview. Aggregate analytics across templates owned by all groups land in Wireframe 23.
