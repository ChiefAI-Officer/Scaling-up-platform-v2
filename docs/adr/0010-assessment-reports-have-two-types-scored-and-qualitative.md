# Assessment per-respondent reports have two types — *scored* and *qualitative* — chosen by an explicit per-template config, not inferred from the scoring shape

The canonical per-respondent Results report (ADR-0007, `BrandedReport`) is a **scored** report: cover → score ring → per-decision/section breakdown → "All sections" score/average table → recommendations. Some instruments are not scored reports at all: the **Leadership Vision Alignment** (LVA) and **Quarterly Session Preparation** (QSP v1/v2) instruments are **prep/discussion surveys** whose value is the respondent's *answers organized by theme* — in Esperto they render as a preface + themed Q&A blocks + a couple of rating tables/bar charts, with **no overall score, no ring, no aggregate "All sections" table**. We therefore split the per-respondent report into **two report *types*** — `scored` and `qualitative` — and select the type per template via an **explicit config map keyed by `AssessmentTemplate.alias`** (`src/src/lib/assessments/report-config.ts`), rather than inferring it from the scoring shape. Unknown/omitted alias → `scored` (current behavior, fully back-compatible). The qualitative type renders **per-respondent answers only** (no team Mean) through a shared data layer feeding both the on-screen/PDF component and the inline-HTML email twin.

## Context

Jeff's June-9 feedback (Spec 17 Wave E, `From Jeff/gabriel-feedback.docx`) asks to remove the score/average tables from Rockefeller (#24), QSP v2 (#27), QSP v1 (#28), and LVA (#30), and to **restructure the LVA report** out of "one long unbroken block" (#31). Reading Jeff's authoritative source — the Esperto-rendered report PDFs in `From Jeff/APP_scaling up assessemnt/` — reveals *why*: LVA and QSP are qualitative prep reports, and forcing them through `BrandedReport`'s scored anatomy produces the wrong report (a "Submitted" headline + "How you scored, section by section" cards for a survey that was never scored). The only place a respondent's actual text answers render today is the **deprecated raw view** (`AssessmentResultView`, retired-phased per ADR-0007) — so the qualitative report is, in effect, the *branded, themed* form of those answers.

Grilled via `/grill-with-docs` (2026-06-17). The pivotal finding: **report-type is orthogonal to the neutral-tier scoring concept (ADR-0002).** A neutral tier exists because `scoringConfig.tiers` requires `min(1)` — it is a *scoring-config* workaround, not a statement about how to *present* the report. **Five Dysfunctions is neutral-tier (ADR-0002) yet must stay a *scored* report** — it exists specifically to show Jeff's per-category totals. So "hide the score table whenever the template is neutral-tier" is wrong: it would strip Five-D's category totals (the thing it's for). Report-type must be decided explicitly per template. Design doc: `docs/specs/v7.6/17e-wave-e-report-polish-design.md`.

## Considered options

- **Explicit per-template config map keyed by `AssessmentTemplate.alias` (chosen)** — a small source-of-truth `REPORT_CONFIG` (`reportType: "scored" | "qualitative"`, `showScoreTable`) that the report loader resolves via the template alias (verified stable across versions; the per-version alias is separate). Unknown alias falls back to `scored` + table, so existing and future templates are unaffected until explicitly opted in. Report-type is engineering policy that changes rarely; a code map is the right home.
- **A principled rule: hide the score table / go qualitative when the template is neutral-tier** — rejected. Five Dysfunctions is neutral-tier but must keep its category-totals table. Inferring presentation from the scoring shape conflates two orthogonal concerns (ADR-0002) and silently breaks the counterexample.
- **Keep one scored renderer; add flags to hide the ring/table for some templates** — rejected. `BrandedReport` never surfaces the respondent's free-text answers at all, so hiding score widgets leaves an *empty* shell for LVA/QSP — it cannot produce the Esperto report. The qualitative templates need a different renderer, not a subtractive one.
- **Store report-type as a column on the template (DB-driven)** — rejected for this wave. Needs a migration + admin UI to be useful, and report-type is not coach-editable product data — it's a platform rendering decision. The config map avoids both with no loss (flipping a template is a one-line change + redeploy).
- **A feature flag gating the qualitative renderer** (the Wave-D pattern) — rejected. Unlike Wave D's send-side lifecycle, this is **pure rendering** with no external side effects. The `REPORT_CONFIG` map *is* the kill-switch (flip an alias back to `scored` and redeploy); rollback is revert + Vercel promote-previous. A flag would add ceremony without benefit.
- **Render the team "Mean" alongside the respondent's answer (as Esperto does)** — rejected for the per-respondent report. The Mean is a cross-respondent team average; computing it is group-aggregation, which is the deferred **group/cohort report** (ADR-0003, ADR-0007, Wave F #22). The per-respondent qualitative report shows **only this respondent's** answers/ratings.

## Consequences

- **Two report types, one canonical report.** ADR-0007 still holds — the per-respondent Results report is *the* canonical view of a completion — but it now has two shapes. `scored` is the existing `BrandedReport` anatomy; `qualitative` is a new renderer (preface → themed Q&A + the respondent's own rating tables/bars). A future reader should **not "add scores back" to LVA/QSP** — the absence of a score ring/table there is deliberate, matching the Esperto instrument.
- **Type is keyed by `AssessmentTemplate.alias`.** Wave E wires `RockHabits` → `scored` (table off), `qsp-v1`/`qsp-v2`/`leadership-vision-alignment` → `qualitative`; `five-dysfunctions`/`scaling-up-full`/`scaling-up-quick` stay default `scored` + table. Adding a template without a config entry safely renders as a scored report.
- **Report-type ≠ neutral-tier.** Neutral-tier (ADR-0002) is a scoring-config concern; report-type is a presentation concern. They do not imply each other (Five-D = neutral-tier + scored). Do not re-couple them.
- **Per-respondent only; the group/Mean view stays deferred.** Consistent with ADR-0003 (LVA group report deferred) and ADR-0007 (individual ≠ cohort report), the qualitative report shows a single respondent's answers — the 1–3 matrix sliders (ADR-0003) render as that respondent's ratings, never a team average. The Esperto group report (team Mean, obstacle %-vote-share) remains a future slice (Wave F #22).
- **Both render paths change together.** The qualitative type ships an on-screen/PDF component **and** an inline-HTML email twin, fed by one shared data-shaping layer — because the Wave-D #15 results-email and #16 coach-notify (live in prod) currently emit the *scored* report for every template; without the email twin, enabling #15 on an LVA/QSP campaign would email the wrong report.
- **Additive, no migration, no flag.** The split is a config module + a `templateAlias` passthrough on the report payload + a new renderer. Content corrections that accompany it (#29 LVA questions, #26 QSP label) are forward-only **DRAFT re-seed + publish** (ADR-0001/0002 version-pinning — existing campaigns keep their pinned version).
- **Scope:** this ADR governs the *per-respondent* report's type split. The public quiz path and invited/public gating (ADR-0008) are unchanged. The deprecated raw view stays retired-phased (ADR-0007); Wave E #21 only polishes its question labels in the interim and does not revive it.
