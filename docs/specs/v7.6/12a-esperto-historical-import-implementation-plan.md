# 12a ‚Äî Esperto Historical Import ‚Äî Implementation Plan

> Implements [12-esperto-historical-import.md](./12-esperto-historical-import.md) (approved, grilled, Codex-hardened). Admin-only, staging-first, TDD. **No code until this plan is approved.** Per-step gate: `CI=true npx next build --turbopack` + targeted tests; failing test first.
> Hardened through **4 adversarial review passes** ‚Äî the spec-level Codex pass + the claudex loop's senior-engineer (R1), security/data-integrity (R2), and ops/SRE (R3) rounds. Every finding's disposition is in the Changelog at the bottom.

## Safety invariants (hold for EVERY step)
- **S1 ‚Äî Additive + *minimal explicit update* only (incl. no destructive UPDATEs ‚Äî R2).** The importer NEVER deletes/truncates and NEVER touches coaches/users/workshops/logins. Critically, even its upserts are constrained: existing rows get **create-only or null‚Üívalue backfill** (e.g. `externalId` onto a hand-created respondent) ‚Äî it **never overwrites an existing non-null field**, and never rewrites an already-imported submission's answers/timestamps unless the source content-hash changed (and then only audited). Enforced by (a) a **restricted transaction client / Prisma mock that throws on any `delete`/`deleteMany`/`updateMany`/`$executeRaw`** ‚Äî **except a narrow allowlist for the exact `pg_advisory_xact_lock` + `lock_timeout`/`statement_timeout` statements the org lock needs (R3); every other raw SQL stays denied, both paths tested** ‚Äî (not just a grep), and (b) field-level update-allowlist assertions.
- **S2 ‚Äî One additive migration + real recoverability.** `AssessmentCampaign.externalId String?` + raw-SQL partial-unique `WHERE externalId IS NOT NULL`. Must pass `scripts/check-migration-safety.mjs`. Before it (and before any import) `npm run snapshot:prod` runs ‚Äî but **its table coverage must first be expanded to the full assessment import chain** (`org_respondents`, `assessment_campaigns`, `assessment_invitations`, `assessment_submissions`, `assessment_campaign_participants`, `audit_logs`) and a **Neon PITR checkpoint** taken (R3 ‚Äî today's snapshot omits these, so the rollback story is currently overstated; see Deploy/rollback ¬ß26).
- **S3 ‚Äî No emails (every send route ‚Äî R2).** Imported invitations are created directly as `SUBMITTED`; the importer never calls the send path. Plus a route-side deny guard `status==="CLOSED" || externalId != null` on **every** route that calls `sendAssessmentInvitationEmail` ‚Äî invite, **resend**, **reminders** ‚Äî each tested (not just the bulk invite route).
- **S4 ‚Äî Don't disturb live.** Never mutate published template versions or the live `org-survey/[campaignAlias]/submit` path. The ONLY shared-code change is a default-off `allowMissingRequired` option on `scoreSubmission`.

## Build order (each = TDD slice)
1. **Migration** ‚Äî add `AssessmentCampaign.externalId String? @unique` (R1: the Prisma `@unique` is **required** so the `upsert` selector compiles), then **raw SQL drops the generated full-unique index and recreates it as partial `WHERE externalId IS NOT NULL`** ‚Äî the exact `Organization.externalId` pattern; run `prisma generate` + typecheck. Store the value **namespaced as `esperto:<campaignid>`** (R1 M5 ‚Äî avoids collision with any future external source). Tests: rejects duplicate non-null, allows multiple NULLs; `check-migration-safety` green; snapshot precedes apply (S2).
2. **`scoreSubmission` option** ‚Äî add `options?: { allowMissingRequired?: boolean }` (default `false`). Tests: `false` is byte-identical to today (lock with the existing suite); `true` routes `missingRequired` ‚Üí `unansweredKeys` instead of throwing (S4).
3. **No-email route guard (all send routes ‚Äî R2)** ‚Äî enumerate every route that calls `sendAssessmentInvitationEmail` (invite, resend, reminders) and add the `status==="CLOSED" || externalId != null` deny guard to each. Test per route: CLOSED/imported campaign ‚Üí 4xx, no send (S3).
4. **Parse/classify** ‚Äî `lib/assessments/esperto-import/{parse,classify,types}.ts`: zod-validate + classify members / report / restricted. Sanitized fixtures committed under `__tests__/.../fixtures/`; raw `From Jeff/Exports/` gitignored.
5. **Crosswalk** ‚Äî `crosswalks/{index,qsp-v2,rockefeller,lva}.ts`: typed `Crosswalk` + variant registry + `locked` gate + exhaustiveness guard (unknown **answer** key ‚Üí hard error; unknown **metadata** ‚Üí warn). QSP v2 map (22 pairs, droppedKeys = Q3_5/Q6a/Q7a/Q8a/Q11a/Q16) `locked:true` only after the ~8 ambiguous orderings are confirmed against the seed screenshots; rockefeller/lva `locked:false`.
6. **Crosswalk‚Üîversion validation** ‚Äî for the pinned latest-published version, assert every mapped `stableKey` exists with matching `type` + `scale` (ADR-0001 makes key+type+scale a complete check; catches `patch-qsp-v2-text-to-slider` drift). Mismatch ‚Üí block.
7. **Roster import** ‚Äî `roster-plan.ts` (field map; filter `testuser`/non-active; `dedupeSource="external"`/`dedupeValue=memberid` with `normalizedEmail` fallback that **backfills** `externalId` but **never overwrites** an existing non-null field ‚Äî R2 S1; one Org per file; company name from filename, editable) ‚Üí preview; `commit.ts` (atomic tx). **Org match/create runs under a transaction-scoped advisory lock keyed by `ownerCoachId+normalizedName`, re-queried inside the tx (R2)** ‚Äî no DB unique exists, so this prevents concurrent duplicate Orgs. Admin route `POST /api/admin/assessments/import {mode:"preview"|"commit"}` (**ADMIN-only** ‚Äî R2; not STAFF, since `User.role` defaults to STAFF; rate-limit) + page.
8. **Results import** ‚Äî `results-plan.ts`: group rows by `campaignid`; resolve target Org via `memberid`‚Üí`externalId` (block if roster absent, skip unresolved); published-version preflight + `resolvePublishedTemplateVersion`; reconstruct campaign (`externalId`=campaignid, `CLOSED`, `INVITED`, backdated `openAt`/`closeAt`); build live-shape `[{stableKey,value}]` answers (slider `0`/blank ‚Üí omit, `allowMissingRequired:true` partial+flag). **Zero-scorable rows are SKIPPED + reported (R2)** ‚Äî NOT imported as neutral submissions, because `getAggregateReport` divides by all submission rows and a neutral row would skew the team mean. Full chain Participant + `SUBMITTED` Invitation (inert token) + Submission; `isCEO` via single-CEO guard ‚Üí preview; commit (upsert: campaign by `externalId`, participant/invitation by `@@unique`, submission by `invitationId`). **Submissions are create-only on re-import** ‚Äî an existing imported submission is rewritten only if the row's source content-hash changed, and the change is audited (R2 S1). **Campaign idempotency is verified, not assumed (R1 M5):** an existing campaign matching the namespaced `externalId` is reused **only if its `(organizationId, templateId)` also match** ‚Äî otherwise **block** (guards against an Esperto `campaignid` colliding across orgs/templates; vendor-global uniqueness is unproven).
9. **Admin UI + upload bounds** ‚Äî upload `.json`; required owning-coach picker; editable company name; preview tables (create/update/skip/block with reasons + a sample answer‚Üíquestion-label mapping); commit button. **Concrete bounds (R3):** reject by `Content-Length` > 5 MB before parsing; ‚â§ 2,000 rows/file, ‚â§ 50 campaigns/file, ‚â§ 200 questions/template, ‚â§ 10,000 chars/string; strict unknown-key policy; `statement_timeout` + `lock_timeout` (~10 s) on the commit tx; **v1 commits one campaign at a time** (canary-sized batches). All parsing/scoring stays in the read-only preview, **outside any write transaction**, so commits hold the lock briefly. **Import-specific rate limiter** with low preview/commit quotas that **fails CLOSED for commit** when the distributed limiter is unavailable (R3 ‚Äî not the default fail-open posture).
10. **E2E + guard tests** ‚Äî roster‚Üíresults on fixtures; re-run = no duplicates (and no field overwrite); + guard suite: no-email **per send route** (invite/resend/reminders), unknown-answer-key block, unpublished-version block, wrong-type pinned-version block, incomplete-row partial+flag, **zero-scorable skip**, **duplicate-within-payload blocks** (memberid / normalizedEmail / reportid / (campaignid,memberid)), **parallel-org-commit ‚Üí single Org** (advisory lock), and **S1 enforced via a restricted/mock tx client that throws on any delete/deleteMany/updateMany/$executeRaw** (+ field-level update-allowlist), not just a grep.

## Edge / unhappy paths
11. Malformed/non-JSON or wrong export kind ‚Üí block with a clear message; a file with zero valid rows ‚Üí block.
12. Report row whose `memberid` isn't in the roster ‚Üí **skip + flag** (never anonymized).
13. Owning coach not in the system ‚Üí operator must pick an existing coach (importer never mints one); none selected ‚Üí block.
14. Duplicate-source identities within one payload ‚Üí **block** (no last-writer-wins collapse, R2): duplicate `memberid`, duplicate `normalizedEmail`, duplicate `reportid`, or duplicate `(campaignid, memberid)`; also a `memberid` resolving to >1 Organization. No silent merge.
14b. **Resolver split ‚Üí block (R1 H2).** If a respondent's `externalId` lookup and `normalizedEmail` lookup resolve to **different** existing rows, or the email-matched row already carries a *conflicting* `externalId`, **block** ‚Äî never guess. Backfill `externalId` **only** when the email row is the sole match and has no existing external identity.
15. Target version DRAFT/unpublished (SU Full, or a fresh/dev env where QSP is DRAFT) ‚Üí block via preflight; crosswalk `locked:false` (rockefeller/lva) ‚Üí results import refused.
16. Esperto value out of band (slider `0` on `min:1`) ‚Üí treated as unanswered (omitted), not a real score.

## Concurrency / timing
17. Two concurrent commits of the same file ‚Üí the unique keys (`externalId`, `@@unique`, `invitationId`) make the second an upsert/no-op; **Org create is additionally serialized by a `ownerCoachId+normalizedName` advisory lock (R2)** since no DB unique covers it.
18. Preview is read-only and may go stale; **commit re-resolves + re-validates inside its own transaction**, and all parsing/scoring happens in the preview **outside** the write tx (R2) so the tx ‚Äî and the advisory lock ‚Äî are held only briefly.

## Data integrity (the user's #1 worry)
19. Commit is **one all-or-nothing transaction per file/campaign**; any mid-commit failure rolls the whole import back ‚Äî never a half-imported campaign.
20. The importer is **additive + non-overwriting** (S1): it cannot delete/wipe coaches/workshops/users/logins, AND cannot silently overwrite an existing non-null field or an already-imported answer (the subtle "destructive-by-update" path ‚Äî R2). The migration is additive-only (S2) with a prod snapshot first. All enforced by the restricted-client + field-allowlist tests.
21. Re-import is idempotent (upsert keys in step 8); back-dated `submittedAt` = the Esperto row date; campaign `openAt`/`closeAt` = min/max row dates.

## Failure-mode UX
22. The preview lists every create/update/skip/block with a human-readable reason; any blocking error prevents commit entirely; partial-field rows are flagged ("imported with N missing fields").
23. Audit row written **inside the commit transaction** via `tx.auditLog.create` (R2 ‚Äî rolled back with a failed import, never silently swallowed): operator, coach, org, **source SHA-256, importRunId, file kind, affected entity IDs, create/update/skip/block counts, redacted error summary**. Rich enough to reconstruct/undo an import.

## Out of scope (v1)
24. SU Full import (parked ‚Äî unpublished + partial crosswalk + privacy-by-data); coach self-service import button; Rockefeller/LVA live import (crosswalk stubs only, pending Jeff's sample exports); the pre-existing committed-PDF PII leak under `From Jeff/` (separate cleanup).

## Deploy, rollback & observability (R3)
25. **Deploy ordering (no-email rollback safety).** Ship the CLOSED/imported send guard (step 3) as a **separate, backward-compatible release BEFORE any import runs**, so even a rollback to "guard present, no import data" stays safe. Once imported campaigns exist, pre-guard deployments are **rollback-forbidden**; add a post-rollback smoke test + alert for any send attempt on an imported/CLOSED campaign.
26. **Recoverability (the data-loss backstop).** Before prod rollout, expand `snapshot:prod` to the full import chain (S2) and take a **Neon PITR checkpoint** immediately before each import. Add an **import-rollback runbook keyed by `importRunId`/`externalId`** ‚Äî operator-run, approval-gated, with exact reverse-dependency cleanup (submissions ‚Üí invitations ‚Üí participants ‚Üí campaign; plus roster respondents created by the run). This rollback is a deliberate operator procedure, **separate from the importer**, which still never deletes (S1).
27. **Flags + canary.** `ESPERTO_IMPORT_ENABLED` and `ESPERTO_IMPORT_COMMIT_ENABLED` default **off**; prod commit gated to an operator allowlist; first prod use is a **canary** (one roster + one campaign) before widening; documented flag-rollback path.
28. **Observability.** Emit structured `assessment.esperto_import.*` logs/metrics (`importRunId`, source hash, mode, counts, duration, skip/block reasons, rewrite count, lock wait/timeout, no-email-guard denials) + alert gates (repeated preview failures, high skip/block rate, **any** no-email denial, **any** changed-source rewrite) wired into the existing `/admin/observability` + runbook (spec 06).
