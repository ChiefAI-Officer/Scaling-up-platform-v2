Topics                             User Roles & Permissions                      Problem:            The current prototype mixes admin and coach functions, creating a confusing user experience.                            Solution:            Implement distinct views for each role.                            Admin:            Full control to create and edit templates for landing pages, offers, and thank you pages.                            Coach:            Uses admin-defined templates to create workshops; has limited editing options.                            Rationale:            This structure ensures brand consistency and reduces the risk of coach-introduced errors.                                   Coach Onboarding & Profile Management                      Problem:            Manually entering coach data is inefficient and risks inconsistency.                            Solution:            Research integrating with Circle.so.                            Authentication:            Use Circle.so for coach login to avoid building a separate user management system.                            Bio Sync:            Add a "Refresh data from Circle" button to pull coach bios and photos.                            Rationale:            This leverages existing data and reduces development complexity. Discount Codes                      Requirement:            Coaches need to create discount codes for workshops. Integrated Surveys                      Problem:            The current Typeform process requires manual duplication of surveys for each event, which is unsustainable for automation. Data Centralization:            All survey responses will be stored in the app's database.                            Rationale:            Centralizing data enables future trend analysis and AI-driven insights.
