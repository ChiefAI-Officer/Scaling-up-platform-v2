# Triage Labels

Every issue carries exactly one **category** label and one **state** label.

## Category labels

| Role | Label | Meaning |
|------|-------|---------|
| bug | `bug` | Something is broken |
| enhancement | `enhancement` | New feature or improvement |
| security | `security` | Security hardening or vulnerability |
| non-code | `documentation` | Config, docs, or manual action required |

## State labels (triage state machine)

| Role in mattpocock/skills | Label in this repo | Meaning |
|---------------------------|--------------------|---------|
| `needs-triage` | `needs-triage` | Maintainer needs to evaluate |
| `needs-info` | `needs-info` | Waiting on reporter for more information |
| `ready-for-agent` | `ready-for-agent` | Fully specified, ready for an AFK agent |
| `ready-for-human` | `ready-for-human` | Requires human implementation or judgment |
| `wontfix` | `wontfix` | Will not be actioned |

## State transitions

```
[unlabeled] → needs-triage → needs-info (→ needs-triage when reporter replies)
                           → ready-for-agent
                           → ready-for-human
                           → wontfix
```

Every AI-generated triage comment must start with:
> *This was generated by AI during triage.*
